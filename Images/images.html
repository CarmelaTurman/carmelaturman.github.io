Using Images in HTML and CSS

Images can be presented in HTML either with an <img> tag or as a background image using CSS.

HTML <img> tag

The <img> tag directly embeds an image into the HTML content. This image is part of the document structure, meaning it can have alt text, captions, and semantic meaning.

CSS background-image

A background image is applied to an element via CSS, and is not part of the documentâ€™s content flow. It is purely decorative, and does not have semantic meaning.

Background Image

Key differences:

  • HTML <img>: part of content, semantic meaning, supports alt text for accessibility, affects layout directly.
  • CSS background-image: purely decorative, does not add to document semantics, can be positioned, repeated, sized independently of element content.

CSS Properties for Background Images

Besides background-image, CSS provides other properties to control background image display. Two important ones are:

  • background-size: controls the size of the background image. For example, cover scales it to fill the element while preserving aspect ratio; contain scales it to fit entirely within the element.
  • background-position: determines the starting position of the background image within the element (example center, top left, or coordinates like 50% 50%).

Dimensions vs. Resolution in Bitmap Images

In one sentence: Dimensions refer to the width and height of an image in pixels, while resolution refers to the pixel density (pixels per inch) that affects image clarity.

Image File Formats

GIF: Supports animation, limited to 256 colors, good for simple graphics and icons, not ideal for detailed photos.

PNG: Supports transparency, lossless compression, ideal for graphics with sharp edges and transparent backgrounds.

JPG/JPEG: Lossy compression, good for photographs, smaller file sizes but quality decreases with compression.

WebP: Supports both lossless and lossy compression, smaller file sizes with high quality, supports transparency and animation.

SVG: Vector format, scalable without losing quality, ideal for icons and logos, not suitable for complex photographic images.
